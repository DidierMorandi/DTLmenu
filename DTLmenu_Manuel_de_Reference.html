DTLmenu — Manuel de référence

Référence descriptive du lanceur de compilation de la suite NetDTL.

Type : HTA WindowsMoteur : mshta.exeCible : suite NetDTL

Rôle

DTLmenu centralise les opérations de build des outils Python de la suite NetDTL dans une interface graphique locale. Il décrit une couche d’orchestration autour de Python, PyInstaller et des fichiers de spécification propres à chaque projet.

Nature HTA

Le choix du format HTA permet d'accéder aux objets COM nécessaires à l'écriture de scripts temporaires, à l'ouverture de dossiers Windows et au lancement de commandes locales. Le document HTML est donc à la fois interface et contrôleur d'exécution.

Chaîne de build

Préparation

Le lanceur vérifie la disponibilité de Python et de PyInstaller, puis prépare les chemins de travail à partir du dossier racine configuré.

Versionnement

`DTLversion.py` sert de composant d'encadrement pour les builds, avec une logique de version et de rollback intégrée au script temporaire.

Sorties

Les exécutables produits sont regroupés dans les dossiers `dist` des projets concernés.

Configuration persistante

La configuration principale est le dossier racine de la suite. Elle est mémorisée localement dans `DTLmenu.root`, ce qui évite de modifier le fichier HTA lors d'un changement d'emplacement.

Limites

  • Application réservée à Windows et au moteur HTA historique.
  • Dépend du rendu IE intégré à `mshta.exe`.
  • Suppose une arborescence cohérente de la suite NetDTL.
  • Ne constitue pas un outil de déploiement applicatif.
