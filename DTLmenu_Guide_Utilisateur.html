DTLmenu — Guide utilisateur

Guide du lanceur graphique de compilation de la suite NetDTL.

Windows HTAPyInstallerSuite NetDTL

Objectif

DTLmenu permet de compiler les outils Python de la suite NetDTL depuis une fenêtre unique, sans ressaisir les commandes PyInstaller à la main.

Exemple d’utilisation

Après une modification dans GitDTL, ouvrir DTLmenu, vérifier le dossier racine, lancer le build GitDTL, puis récupérer l’exécutable généré dans le dossier dist du projet.

Préparer l’environnement

  1. Vérifier que Python est installé.
  2. Vérifier que PyInstaller est disponible.
  3. Vérifier que les projets de la suite sont présents dans le même dossier racine.

Exemple de préparation

Avant de compiler DTLaudit, ouvrir une invite de commandes et contrôler que python --version et pyinstaller --version répondent correctement. Vérifier ensuite que les dossiers DTLaudit, GitDTL et les autres projets NetDTL sont rangés sous le même répertoire parent.

Définir le dossier racine

Le champ ROOT_DIR doit pointer vers le dossier qui contient les projets de la suite. Une fois validé, DTLmenu mémorise ce chemin dans DTLmenu.root.

Exemple de dossier racine

Si les projets sont dans C:\Users\Utilisateur\Documents\Secato\outils, indiquer ce chemin dans ROOT_DIR. DTLmenu pourra alors retrouver, par exemple, C:\Users\Utilisateur\Documents\Secato\outils\GitDTL ou C:\Users\Utilisateur\Documents\Secato\outils\DTLaudit.

Si un projet n’est pas trouvé, commencer par vérifier le dossier racine plutôt que les commandes de build.

Choisir un build

Build individuel

Chaque outil Python peut être compilé séparément : DTLknowsWhy, DTLsaysWhat, GitDTL, GitHubMenu, DTLaudit, DTL4u, DTLarchive ou DTLi18n.

Exemple : après avoir corrigé uniquement DTLi18n, choisir le build DTLi18n pour générer seulement cet exécutable, sans relancer toute la suite.

Build complet

Le build complet enchaîne les compilations principales de la suite, y compris DTLi18n.

Exemple : avant de publier une nouvelle version interne, lancer le build complet pour régénérer tous les exécutables Python attendus et vérifier que chaque projet compile encore correctement.

NetDTL

NetDTL est un projet PHP/MySQL : il n’a pas besoin de compilation PyInstaller.

Exemple : si la modification concerne uniquement les fichiers PHP de NetDTL, utiliser DTLmenu comme point de repère documentaire, mais ne pas lancer de build PyInstaller pour ce projet.

Où trouver les résultats

Les exécutables générés sont placés dans les dossiers dist des projets concernés. DTLmenu propose des accès rapides vers ces dossiers.

Exemple de récupération

Après un build GitDTL réussi, ouvrir le raccourci vers le dossier de sortie et chercher l’exécutable dans GitDTL\dist. Le fichier obtenu peut ensuite être copié vers l’emplacement de distribution habituel.

Dépannage courant

  • Si PyInstaller manque, installer ou mettre à jour PyInstaller. Exemple : exécuter python -m pip install --upgrade pyinstaller, puis relancer DTLmenu.
  • Si un build échoue, vérifier les dépendances Python du projet concerné. Exemple : pour GitHubMenu, installer les modules signalés dans le message d’erreur avant de relancer le build GitHubMenu.
  • Si le dossier dist n’apparaît pas, vérifier que la compilation est terminée sans erreur. Exemple : relancer le build individuel et lire le journal affiché par DTLmenu jusqu’à la fin.
  • Si DTLmenu ne trouve pas les projets, corriger ROOT_DIR. Exemple : remplacer un chemin pointant vers ...\DTLmenu par le dossier parent qui contient tous les projets de la suite.
